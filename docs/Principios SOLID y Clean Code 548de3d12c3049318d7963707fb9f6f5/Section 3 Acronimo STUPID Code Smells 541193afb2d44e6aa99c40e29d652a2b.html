Porque el patron singleton es un code smells?

pros:

Contras

Bajo acoplamiento y buena cohesion

En nuestro desarollo tenemos que buscar un bajo acomplamiento y buena cohesion

desventajas:

  • El ensamblaje de modulos puede requerir mas tiempos
  • Los modulos llegan a ser dificil de reutilizar y/o probar porque se deben incluir modulos dependientes

Cohesion:

  • La cohesion se refiere a lo que la clase (o modulo) puede hacer.
  • La baja cohesion significaria que la clase realiza una gran variedad de acciones, No se enfoca en lo que debe hacer.
  • Alta cohesion significa que la clase se enfoca en lo que deberia esta haciendo, solo metodos asociados a la intencion de la clase

Codigo no probable

Es codigo dificil de testear:

  • Producto de Codigo con alto acoplamiento
  • Codigo con muchas dependencias no inyectadas
  • Dependencias en el contexto global (tipo singleton)

Optimizaciones prematuras

Complejidad esencial:

La complejidad es inherente al problema

Complejidad accidental:

Cuando implementamos una solucion compleja a la minima indispensable

Nombres poco descriptivos

  • Nombre de variables mal nombradas
  • Nombres de clases genericas
  • Nombres de funciones mal nombradas
  • Ser muy especifico o demasiado generico

Duplicidad del codigo:

No aplicar el principio DRY

Real:(mala)

  • Codigo es identico y cumple la misma funcion
  • Un cambio implicaria actualizar todo el codigo identico en varios lugares
  • Incrementa posibilidades de error humano al olvidar una parte para actualizar
  • Mayor cantidad de pruebas innecesarias

Accidental: (Buena)

  • Codigo luce similar pero cumple funciones distintas
  • Cuando hay un cambio solo hay que modificar un solo lugar
  • Esta duplicidad se puede manejar con refactorizaciones u optimizaciones

Acopladores:

Feature Envy

Esto es cuando un modulo clase o funccion hace excesiva referencia a una funcion de otro modulo

Intimidad inapropiada

Es cuando una clase o metodo usa campos internos de otra clase.

Cadenas de mensajes

es una cadena de llamadas de funciones, una funcion A que llama a B que llama a C, esto se soluciona con comunicacciones directas entre estas para no tener que pasar por puentes de funciones o metodos para poder llegar a la funcion deseada
